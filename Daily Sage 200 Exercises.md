These are some exercises that you can do daily to keep your Sage 200 knowledge fresh. When starting out with Sage 200, try and do the beginner exercises **without help from online/anyone else**, by simply guessing which buttons to click. It is the guesswork and trial and error which will build your confidence as opposed to following a guide. If you get stuck you can then [consult the documentation](https://desktophelp.sage.co.uk/sage200/professional/Content/Home.htm) or use google/chat GPT.   


Before you start these exercises, have a hyper ready and make sure that the financial period for the current date is open (ask someone to help if you don't know what that means). As you do them, it's very important to understand from the customer's point of view **what each feature means**. Eg don't just fill in any value for the VAT code, ask yourself what is a VAT code? Check Google/Chat GPT if you don't know.  


Try and practice a little (eg 15 mins) each day. **Little and often is better** than large amounts of practice every few weeks.  


Unless stated otherwise or unless you want to, only fill in the minimum info necessary to complete each task. When looking at a module for the first time, it's a good idea to look at the **process map** for that module, which under that module in the menu bar. You don't need to fully understand this map, but it helps put things into perspective as you learn more about Sage 200\.  


  




| **Exercise** | **Level** | **Modules** |
| --- | --- | --- |
| Add a customer | Beginner | SL |
| Invoice a customer for £100 \+ VAT | Beginner | SL |
| Record a receipt from the customer for £100 \+ VAT | Beginner | SL/CB |
| Allocate that receipt to the invoice | Beginner | SL |
| Add a supplier | Beginner | PL |
| Record an invoice from the supplier for £100 \+ VAT | Beginner | PL |
| Record a payment to that supplier for £100 \+ VAT | Beginner | PL/CB |
| Allocate the payment to the invoice | Beginner | PL |
| Add a bank account (hint: look at other bank accounts for what values you should enter) | Beginner | CB |
| Enter a sales order | Beginner | SL |
| Allocate stock for the sales order (if it wasn't done automatically) | Beginner | SL |
| Despatch the sales order | Beginner | SL |
| Print the invoice for that sales order | Beginner | SL |
| Post the invoice for that sales order. Posting here means "Creating and saving permanently" (nothing to do with posting by mail!). Look for your invoice in the invoicing list. This invoice will also be created in the sales ledger, can you find it? What is the difference between the one in the sales ledger and the one in SOP? Does one provide more detailed info than the other? | Beginner | SL |
| Add a stock item | Beginner | Stock Control |
| (Take a checkpoint first, then apply it after). Open and close an accounting period | Intermediate | Accounting System Manager |
| (Take a checkpoint first, then apply it after). Do a "year end" by closing all accounting periods in the year and closing the financial year, which should open up the accounting periods in the next year. | Intermediate | Accounting System Manager |
| Look for all customers that are on hold (hint: add a filter in the customers list) | Intermediate | SL |
| Set up ageing periods, then print and view the aged debtors report. What does it mean? | Intermediate | SL |
| Add a warehouse and bin | Intermediate | Stock Control |
| Move stock items from one warehouse to another | Intermediate | Stock Control |
| Choose a module, look at the process map. Select a step you either haven't seen before or are rusty on, and refresh yourself on that step, trying to actually perform some action rather than just look at the form (eg actually add a customer etc). Understand why the user would want to perform this step. | Advanced |  |
