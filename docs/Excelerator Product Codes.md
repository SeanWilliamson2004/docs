**Product Codes** The purpose of this page is to specify how the procedure for creating new Codis Product Codes.   

## Background

Every product sold by Codis have a unique product code. The products which require the most planning are Codis Products in general and Excelerator Modules in particular.   
The product code is used in Development, CRM, Sage 50 with plans to have the product code used in the web sites, brochures etc.   
Each Excelerator module will have four or six (including Enterprise) product codes. 

1. Standard Product
2. Standard Product Licence and Maintenance
3. Thin Client Product
4. Thin Client Product Licence and Maintenance
5. Enterprise Product
6. Enterprise Product Licence and Maintenance

## Where the Product Code is used

The product code needs to be agreed before development can release a product. There is an argument that says a product code needs to exist much earlier in the cycle \- when we start market research or/and start to record customer interest in a specific module. 

### Spread Sheet

A central spread sheet is used to control all product codes. This spread sheet is available at this [file location](https://codislimited.sharepoint.com/sites/Wiki/Sales/Sales%20Wiki/Documents/Licences%20Database/2013%20new%20version/Testing%20-%20before%20release/Product%20Codes/Product%20Codev1.8.xlsx). The spread sheet records the product details and characteristics (e.g. the version of Sage, whether standard or Licence and Maintenance (L\&M) etc. The spread sheet records both a long and short name along with a unique product code. ### CRM

The CRM records customer interaction. Having the Product Code in the CRM from an early stage allows sales to record customer interest in specific modules.   
The Licensing is ran from within CRM. The Licensing needs to have the Product Code. However Licences will not be recorded until product is released. 

### Sage 50

Sage 50 is used to produce all delivery notes and invoices for this reason the product code needs to be stored in Sage 50\. 

### Web Site

Any marketing materials and downloads should also quote the product code. The Single install setup will have an impact here. 

### Brochures and Flyers

For Sales to fill out 

## Stages of Creating Product Code

New Codis modules will go through various stage and the product code needs to follow this cycle. 

### Planning

This is when we consider possible modules. Each module needs to be recorded and discussed as to the possible customer interest and the merit on whether to develop the module or not. 

### Market Research

Possible modules need to be researched to gauge the potential sales for the product. Suggest we have a Research group \- possibly consisting of some Business Partners, some accountants, Sage (or producer) product managers etc. Final decisions rest with Sean and Mohan. 

### Road Map

This will schedule development, release dates, marketing plans. 

### Development

The product is developed \- any module specific benefits, features need to be well thought out and investigated before development starts. 

### Release

Before the module is released it needs to go through a thorough testing process. be included in the download area of the website (Single setup?). have any specific Help pages to be written up. have all Sample Excelerator Templates designed accessible for development. 

### Launched

The product is launched to the Codis Community.
