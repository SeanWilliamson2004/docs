This is an FAQ for external logins with the new licensing system, with a view of using this for our online help as well as the Training Manual. Please note that a separate page has been created for the benefits of using a Google / Microsoft Account. 

Page created to reflect and pre\-empt possible Frequently Asked Questions (FAQS). 

## Intro

The new Codis Licencing System allows our customers to quickly and efficiently licence their Codis products / services such as the Codis excelerator. This eliminates the need to contact Codis every time you want to licence your product. 

The New licencing system uses External Logins (A Google / Microsoft Account) to quickly update and licence products / services. 

## What is External Login?

External login is a form of single sign\-on that uses existing login information from a Google or Microsoft Account to log in to a third\-party system (in this case the online licencing system for Codis ) in lieu of the user creating a new login specifically for that website. 

It is designed to simplify the login process. 

## Do I have to create a new email address?

**No**. You can use your work email address (existing email address) to create a Google / Microsoft Account. A Google / Microsoft Account is not the same as a Gmail, Windows Live ID or Hotmail account. 

Links to create a new account without a new email address: 

1. Google: [https://accounts.google.com/signupwithoutgmail](https://accounts.google.com/signupwithoutgmail)   
[Guide on creating a Google Account](http://intranet/wiki/5116)
2. Microsoft: [https://signup.live.com](https://signup.live.com/)

## Do we have to create a Google / Microsoft Account for every user?

**No.** Only the *Licence Admin* will need one. The Licence Admin can: 

1. Log on using external logins
2. Licence Codis products
3. Control which users use which products by logging onto our [licence portal](https://www.codis.co.uk/licence)

## Why does Codis use a Google Account or Microsoft Account for the online Licencing System?

There are many benefits for the user in using a Google / Microsoft Account for licencing their Codis products. 

External Logins are the fast, easy and secure way to sign in to websites. 

## Why can't I create a Codis Account instead of using a Google or Microsoft Account?

Shorten

External Logins provide a method for users to grant third\-party access to their resources without sharing their passwords. It also provides a way to grant limited access in scope, duration etc. 

A lot of third party sites and systems are moving towards using external logins and this will be of great benefit to Customers. 

Codis believes that this is the way forward and will provide extra security for our customers. 

Creating user logins and passwords for Third Party sites instead of using an External Login (using a Google / Microsoft Account) can present a greater risk. Not only does this require exposing your password to someone else but also presents other security risks with regards to sensitive data. 

Because the focus of most External Login providers (such as Google and Microsoft) is in identity management, they can be more thorough about protecting your online identity. 

Most website operators are less likely to be as dedicated to protecting your identity as the External Login Providers, whose focus is on securely hosting user identities. 

It is a safer and more secure way for users to interact with protected data. 

## How does it improve security?

### A summary of the Security Benefits:

For our customers – the major benefit is security. 

- The technology and authentication frameworks used by External Login providers such as Google and Microsoft increase security of user data as their focus is Identity management and the integrity of user identity verification is maintained by the most expert service providers.
- The top external login providers employ state of the art security systems and full time security teams. This sophisticated technology analyses every sign in attempt in real time, taking into account the user's previous behaviour, the reputation of the IP address as well as the geographical location of the sign\-in attempt. It also allows users to view their current sessions as well as giving users the ability to sign out of sessions remotely. Additionally – alerts are sent out to users when unusual activity is detected on their account.
- No user name or password is transmitted during the third\-party authentication process, only authorisation tokens. This means that at no stage is your password shared with Codis.
- Multi\-factor authentication further improves security. External login providers such as Google also give users the option to enable multi\-factor authentication on their accounts. Signing in with a username and password results in a numeric code being sent to the user's mobile phone. This is the same Industrial strength security feature used by financial websites and enterprise applications.

## Here are just a few benefits of using External Logins.

### Minimize Password Security Risks

Many web users deploy the same password across multiple websites. And since traditional passwords are not centrally administered, if a security compromise occurs at any website you use, a hacker could gain access to your password across multiple sites. With External Logins, passwords are never shared with any websites or systems, and if a compromise does occur, you can simply change the password for your External Login, thus immediately preventing a hacker from gaining access to your accounts at any websites you visit or systems that you use. 

Working with protected data \- users shouldn't be spreading their passwords around the web to get access to it. Using External Logins allows Codis to give users access to their data while protecting their account credentials. 

### Gain Greater Control Over Your Online Information

Using External Logins is a decentralized standard, meaning it is not controlled by any one website or service provider. You control how much personal information you choose to share with websites or online systems that accept external logins – such as Codis. Using External Logins allows you to use that portable identity across the web. 

### Reduce Frustration Associated with Maintaining Multiple Usernames and Passwords

Most web users struggle to remember the multiple username and password combinations required to sign in to each of their systems, and the password recovery process can be tedious. But using the same password at each of your favourite websites poses a security risk. With External Logins, you can use a single account (from providers like Google or Microsoft) to sign in to different systems, websites or online services without ever needing to create another username and password. External Logins is the safer and easier method to joining new sites, services or systems. 

### Accelerate Sign Up Process across different web services and systems.

Most websites ask for an extended, repetitive amount of information in order to use their application. External Logins accelerate this process by allowing you to sign in with a single click. Once again – this is done without sharing or transmission of user names or passwords meaning that your information and data is safe. 

## We created a Google Account using the work email of a user – but what if that User leaves our company?

Just like you would create a new email account for a new starter – create a Google / Microsoft Account for the new user who will be the Licence Admin. Contact Codis and we will update our information and you can continue to use the online licencing system as before using the external login account for the new Licence Admin. This process will be quicker and simpler as you will not need to contact Codis every time you want to Licence Codis products such as the Excelerator.
