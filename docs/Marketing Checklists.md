## **Tutorial Video Editing Checklist**

- When beginning the editing process, ensure you have access to the temporary audio. It will help maintain the correct length for different shots and enable you to compose the entire video in the best way possible
- Every video will begin with Codis' branding animation.
- A one\-line introduction to the video's subject matter goes before the Codis' logo animates.
- The Codis' logo animates with a bell sound.
- When showing an excel sheet shot in a video, make sure a white strip is there at the bottom. This strip is opaque on the right side and transparent on the left side.
- Keep the background music as low as it doesn't interfere with the voiceover at all, but is still hearable throughout the video play.
- Zoom in where possible to make the video small\-screen friendly.
- When zooming in or zooming out, don't use any transitions. Use cut to zoom and then cut to normal, instead.
- Don't exceed the duration of a video of more than 5 minutes.
- End\-screen animation with sound is available as pre\-made. There is no need to re\-create the end\-screen animation for every video.
- Getting the professional voice\-over for the end\-screen isn't recommended. We have a voice\-over for the end screen readily available. We can reuse it in every video. This helps us keeps the voiceover order short in terms of word length.

## **Blog posting checklist:**

- Compose a blog as a draft
- Add a link of the related published youtube video to the post
- Add a link on blog's listing page to the blog post page
- Publish the blog post before the YouTube video is published and made public
- Take the video link and add it in the blog page
- Take the link of this blog post and put it in the related video description so that the video can be published.
- Share the post on social platforms like Twitter, Linkedin and Facebook.
- While preparing a post to share on socail platforms, search for the related \#tags and add them to the post

## **Mailer checklist**

**Splash screen mailer**

- The splash screen mailer is the main mailer that will be published For Excelerator  every 3\-4 months
- Because this mailer is for Excelerator only, it should not be accessed by search engines. Keep the search engine's access to this page blocked or off.
- The splash screen that appears when users launch it for the first time, has been designed as a web page on the website which is based on the Squarespace CMS.
- The mailler does not have the website's header and footer as they have been hidden.
- The mailer will not show up to the public directly but via Excelerator. Whenever a mailer will be pushed, it will appear on the Excelerator's launch.
- The mailer will not appear in the next launch of Excelerator. (The splash screen pop\-up appears only when the splash screen page is updated. Ideally, the page will be updated when new news items will be added. This system in Excelerator software automatically detects that the page has been updated. Now, whenever users launch the Excelerator software from Excel, the popup appears. It won't appear again till the splash screen web page is not updated).
- Whenever the mailer page will be updated with new content, then the mailer will appear again on the Excelerator's launch.
- The page and its URL will never change for the mailer. The content will change, instead.
- When pushing new content or updating the mailer with new content, please draft it on the replica page. Get the approval and then put its code on the main mailer page.
- Add tracking code to every URL added to the page.

**Newsletter blog post**  

- The newsletter will be published along with the publication of splashscreen mailer. It will have the same news items of the splashscreen mailer. But it will be a full\-page design a like a nomral web page.
- If you are duplicating an older blog post and then thinking to update its items to avoid extra work then make sure you also change its description and excerpt.
- When a new post with old description and excerpt is tried to shared on twitter and linked, then these platform will pick up the description on old post. Now, event after updating the description and excerpt with new one, the linkedin and twitter will continue to show the new post with old description or exceprts. It happens because whenver but an URL on social platform, they immediately create a capture of image, description and title and other meta things that present on a web page at the time it's shared. Now if you want to change meta description, the socail platforms will continue to pick the old stuff. So make sure you make changes in description and excerpt befoe sharing it on social platfroms.

**E shot**  

- The eshot will be sent along with the publication of splashscreen mailer. It will have the same news items of the splashscreen mailer. But it will be designed in mail chimp like a mailer.
- **E\-shot sending checlist**  

When sending an e\-shot, there will be certain things we will need to take care of in order to make the sending completely faultless. Let us understand these things in three stages of sending an eshot.
- **Before**
- Make sure that the design is completely matching with the design of the newsletter blog post which itself follows the design of the splash screen mailer.
- Double\-check all linkings. The thumbnail, title and description and read more button of a single news item must be linking to a single news item, and check carefully that all four linking to single news items are working.
- Make link trackable which means all links must be trackable in the analytics application. Use https://ga\-dev\-tools.web.app/ga4/campaign\-url\-builder/ to make every link trackable effortlessly.
- Make sure that we input the best possible information in Campaign URL Builder so that we can easily identify our clicked links in analytics.
- Cross check all the trackable URLs in analytics
- Send test emails and get approval

- **During**
- When sending an eshot, click on the drop down aside the edit button and choose the create campaign option   
![Screenshot 2022-05-19 220006.png](images/Marketing_Screenshot_2022-05-19_220006.png)
