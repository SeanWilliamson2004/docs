The Release History report will generate a CSV file holding information about which work items are fixed in each release, going back to the beginning of 2018\.

The report ignores abandoned builds, so it important that builds that do not get released in any way are abandoned so the fixes in a release are more easily viewed.

The report can be run by unzipping the file below then running ReleaseHistoryReport.exe.  This will produce a file ReleaseInfo.csv in the same folder.  The csv file can be opened in Excel to tidy.

[ReleaseReport.zip](https://codislimited.sharepoint.com/sites/Wiki/Development/ReleaseReport.zip)

The code is in the ReleaseHistoryReport repository.
