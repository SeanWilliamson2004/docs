## Overview

Our software offers complex functionality and is a highly integrated across set of multiple modules and projects. Coded units are reused across these modules and products to allow efficient and consistent development. 

With this high degree of integration and reuse comes an added risk of further development breaking existing functionality. 

Having to repeat manual complete test cycles to ensure this does not happen is not feasible. Developing a high degree of automated testing helps guard against this problem. Automated tests can be optimal even for testing changes as they are developed. 

## Basic Methodology

The basic structure of any automated test is to run the method or methods being tested, then to *Assert* the expected result. If the assertion fails, then the test will fail
